====== mysql ======
Created 수요일 28 10월 2020

1. Like 구문 %검색%의 경우 앞의 %가 있을시에만 인덱스를 타지 않음.
-> 해당 방법의 대체안은 INSTR(컬럼이름, 값) > 0;

* INSTR(비교할 대상, 비교하고자하는 값, 비교를 시작할 위치, 검색된 결과의 순번)

2. PK, FK의 경우 해당 값에 대한 키가 자동으로 생성이 된다.
3. WHERE A=1 AND B=2를 조회하고, A B 각각이 단일 인덱스가 생성되어있을 시에
  Oracle의 경우는 전부 다 타지만, mysql의 경우는 인덱스를 하나만 골라서 탄다.
-> mysql의 경우는 결합인덱스를 전부 지정해 주어야함


4. ORDER BY A; -> A 인덱스만 탄다
   WHERE B='' ORDER BY A ->  B 인덱스만 탄다
   -> 위의 경우는 결합인덱스(A,B)를 생성시키고, WHERE 조건에 A 조건을 지정해야한다.

5. BETWEEN은 절도함수이며, 인덱스를 타지않는다
-> 절도함수는 뭐지?? 인덱스를 타지않는다만 기억!
-> MYSQL 특성 같음
-> 날짜 검색시는 WHERE DATE >'2020-10-28' AND DATE < '2020-10-30' 이런식으로 해야 인덱스를 탄다

6. 아무리 인덱스를 탄다고 하더라도 DATETIME형식보다는 INT으로 넣은경우가 속도가 훨씬 빠르다.
-> 앵간한 날짜형태는 INT형으로 넣고, 서버단에서 체크를 해주는게 필요한 것 같음.
