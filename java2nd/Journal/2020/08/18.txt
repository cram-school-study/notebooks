클래스

기본 생성자
-> __init__ 함수

캡슐화 
- 자바의 접근제한자 public, private..
- 데이터와 알고리즘을 하나로 묶어 공용 인터페이스만 제공하고, 세부사항을 감추는 것
- 지정된 객체의 변수를 지정하고 쓰는것을 제한하는것 
( ex) 자바의 name변수 같은 경우 getname() setname()
		파이썬의 __변수 지정하여 get, set 함수 지정,  __변수로 지정시 객체.변수로 사용 불가능 )
- 의무는 아니고 캡슐화가 필요한 클래스에 한해서 사용하면 됨.

상속
- 부모 클래스의 메소드를 그대로 이어 받아 사용하려면 자신의 메소드 이름 앞부분에 super().을 붙이고 매개변수에서는 self를 빼고 사용한다.

인스턴스를 속성으로 사용하기
- 큰 클래스의 일부분을 잘라내 별도의 클래스로 만들어 사용한다.

다형성(Pylymorphism)
- 자바 다형성 - 정적다형성
- 파이썬 다형성 - 동적다형성

클래스를 정의할 때 class ClassNmae:과 class ClassNmae(Object):는 동일하다.

Object 클래스
- 모든 클래스의 최상위 클래스

__new__() 메소드는 객체가 생성될 때 자동으로 호출된다.
__init__()는 그 다음으로 실행된다.

__new__() -> __init__()


__str__() 메소드는 객체의 클래스 이름과 객체의 메모리 주소로 구성된 객체에 대한 설명을 문자열로 반환한다.
