자바

메소드 호출 - 객체 내부에서 호출 - 바로 함수 호출가능
			  객체 외부에서 호출 - new로 해당 객체 생성 후 객체를 통해 호출

메소드 오버로딩 - 클래스 내에 같은이름의 메소드를 여러개 선언.
						 - 조건 : 매개변수의 타입, 개수, 순서가 다름
				- 리턴타입, 매개변수 명은 달라도 상관없음.

인스턴스 멤버와 this
인스턴스 멤버 - 객체를 생성 한 후 사용할 수 있는 필드와 메소드
					 - 각각 인스턴스 필드, 인스턴스 메소드 라고 부른다.

-> 객체 외부에서 인스턴스 멤버에 접근하기 위해 참조변수를 활용하는 것처럼, 
	객체 내부에서도 인스턴스 멤버에 접근하기 위해 this를 사용할 수 있다.

정적 멤버와 static
정적 멤버 - 클래스에 고정된 멤버로서 객체를 생성하지 않고 사용할 수 있는 필드와 메소드
			  - 정적 필드, 정적 메소드로 구성되어 있다.
			  - 정적 멤버는 인스턴스(객체)에 소속된 멤버가 아니라 클래스에 소속된 멤버이기 때문에 클래스 멤버라고도 한다.
		 - static 키워드를 붙여서 선언한다. (정적 필드, 정적 메소드 동일)
		- 클래스에 고정된 멤버이므로 클래스로더가 클래스를 로딩해서 메모리 영역에 적재할때 클래스별로 관리한다. 클래스 로딩이 끝나면 바로 사용 가능. (메소드 영역에 클래스 별로 적재된다.)
		- 클래스이름.필드 , 클래스이름.메소드로 접근한다. (객체 참조변수로도 접근이 가능하나, 클래스 이름으로 접근하는것이 보통이다.)

정적 초기화 블록 - 정적필드는 선언과 같이 초기값을 주는것이 보통이지만, 계산이 필요한 경우를 위하여 자바에서는 정적 블록을 제공한다.
				- 클래스가 로딩될 때 선언된 순서대로 실행된다.

* 정적 메소드와 블록 선언시에는 정적 영역을 먼저 읽어내기 때문에, 인스턴스 멤버,필드가 사용이 불가능하다.(this 키워드 포함)

* 정적메소드와 블록에서 인스턴스 멤버를 사용하고싶으면 객체를 먼저 생성 후, 참조변수로 사용해야한다.


싱글톤 - 전체 프로그램에서 단 하나만 생성할 수 있는 객체를 싱글톤이라고 한다.
		   - new 연산자를 사용할 수 없게 생성자를 private 접근제한자를 붙인다.
		   - 자신의 타입인 정적 필드를 하나 선언하고, 자신의 객체를 생성해 초기화한다.
		   - 정적 필드도 private를 붙여 외부에서 필드값을 변경하지 못하도록 막는다.
		   - 외부에서 호출할수있는 정적메소드인 getInstance()를 선언 후, 자신의 객체를 리턴한다.
